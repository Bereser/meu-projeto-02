Desafio 02- Contribuindo para um projeto Open Source no Github

Projeto Open Source
- É um projeto de código aberto, que pode ser utilizado, modificado e
ditribuído livremente

Como contribuir em um Open Source

- Vá em um repositório aberto
- Depois vá em "Issues", e clique em "New Issues" 
- Vá em uma das opções, mais provavel que você vá seguir o "Feature request"
- Escreva sua sugestão de ideias em frente de "[SUGESTÃO]"
- Depois escreva suas sugestões logo abaixo 
- Logo após isso clique em "Submit new issue"

Como utilizar a contribuição de uma Issue

- Vá em "Issues"
- [Opcional] Comentar no Issue
- Para resolver o problema você irá para "Edit this file" (O Lápis perto do centro direito)
- Depois de resolver o problema você vai em "Commit changes"
- Depois você salvará no final do "Commit message" (fix #1) porque fechara 
a primeira issues que estava em sugestões (E para fechar em diante basta usar #2,#3...)
- Aí clique em "Commit changes"
 
Como adicionar novos templates 

- Vá em "Settings"
- Role pra baixo até em "Features", abilite os "Issues" e clique em  "Set up templates"
- Clique em "Add template: select", depois clique em "Custom template"
- Depois clique em "Preview and edit"
- Clique no lápis (Edit)
- E descreva o seu Issue
- Clique em "Porpose changes"
- Depois clique em "Commit changes"

Como criar uma contribuição

- Clique em "Fork" (no canto superior direito)
- [Opcional] mudar o nome do repositório e adicionar a descrição
- Clique em "Creating fork"

Como clonar um repositório pelo Fork

- Clique no botão verde "Code"
- Copie o código de sua preferência (Ex: https)
- Depois abra o Git bash

Depois de abrir o Git Bash

- git clone linkdocode = para copiar a pasta para o seu computador
- cd nomedapasta = para entrar na pasta remota
- git remote add upstream linkdocode = copia o repositório original
- git remote -v = mostra as pastas copiadas pelo git clone e git remote add upstream
- git pull upstream main = copia a e mescla as alterações feitas pelo git hub ao repositório loca 
- git pull origin main = outra opção de alterar o repositório
- git checkout -b .../nomedeusuario = alterar a branch para outra nova
- touch nomedapasta/nomedoarquivo.md = criar o arquivo com o mesmo nome de usuario do Github
